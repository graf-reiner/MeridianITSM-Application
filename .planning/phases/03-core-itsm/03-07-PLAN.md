1. **Create `apps/api/src/services/notification.service.ts`**:

   Import `prisma` from `@meridian/db`, create a local Queue instance for email-notification (same pattern as billing routes — `new Queue(QUEUE_NAMES.EMAIL_NOTIFICATION, { connection: { host, port } })`).

   **Interface**:
   ```typescript
   interface NotifyPayload {
     tenantId: string;
     userId: string;
     type: string; // NotificationType enum value
     title: string;
     body?: string;
     resourceId?: string;
     resource?: string; // 'ticket', 'change', etc.
     emailData?: {
       to: string;
       templateName: string;
       variables: Record<string, string>;
     };
   }
   ```

   **Export `async function notifyUser(payload: NotifyPayload): Promise<void>`**:
   - Create in-app Notification record via `prisma.notification.create({ data: { tenantId, userId, type, title, body, resourceId, resource } })`
   - If `payload.emailData` is provided: enqueue email job via `emailNotificationQueue.add('send-email', { tenantId: payload.tenantId, to: payload.emailData.to, templateName: payload.emailData.templateName, variables: payload.emailData.variables })`

   **Export `async function notifyTicketCreated(tenantId: string, ticket: Ticket, creatorId: string): Promise<void>`**:
   - If ticket has assignedToId AND assignedToId !== creatorId: notify assignee with type TICKET_ASSIGNED
   - Lookup assignee email for email notification

   **Export `async function notifyTicketAssigned(tenantId: string, ticket: Ticket, assigneeId: string, actorId: string): Promise<void>`**:
   - Notify new assignee with type TICKET_ASSIGNED, title "Ticket TKT-{number} assigned to you"
   - Include email data with `ticket-assigned` template

   **Export `async function notifyTicketCommented(tenantId: string, ticket: Ticket, comment: any, actorId: string): Promise<void>`**:
   - If PUBLIC comment: notify ticket requester (if !== actorId) with type TICKET_COMMENTED
   - Notify assignee (if !== actorId) with type TICKET_COMMENTED
   - Include email data with `ticket-commented` template

   **Export `async function notifyTicketResolved(tenantId: string, ticket: Ticket, actorId: string): Promise<void>`**:
   - Notify requester with type TICKET_RESOLVED
   - Include email data with `ticket-resolved` template

   **Export `async function notifyTicketUpdated(tenantId: string, ticket: Ticket, changes: string[], actorId: string): Promise<void>`**:
   - Notify assignee (if !== actorId) with type TICKET_UPDATED

   **Export `async function getNotifications(tenantId: string, userId: string, filters: { unread?: boolean, page?: number, pageSize?: number })`**:
   - Fetch Notification where tenantId + userId, optional isRead filter
   - Order by createdAt desc, paginate
   - Return `{ data, total, unreadCount }` (unreadCount is always returned for badge display)

   **Export `async function markRead(tenantId: string, userId: string, notificationId: string)`**:
   - Update Notification where id + tenantId + userId: `{ isRead: true, readAt: new Date() }`

   **Export `async function markAllRead(tenantId: string, userId: string)`**:
   - Update many: `prisma.notification.updateMany({ where: { tenantId, userId, isRead: false }, data: { isRead: true, readAt: new Date() } })`

2. **Create `apps/api/src/routes/v1/notifications/index.ts`**:

   **GET `/api/v1/notifications`** — List user's notifications
   - Query: `unread` (boolean), `page`, `pageSize`
   - Call `getNotifications(tenantId, userId, filters)`
   - Return `{ data, total, unreadCount }`

   **GET `/api/v1/notifications/unread-count`** — Get unread count only (for badge)
   - Return `{ count: number }`

   **PATCH `/api/v1/notifications/:id/read`** — Mark single notification as read
   - Call `markRead(tenantId, userId, params.id)`
   - Return `{ success: true }`

   **PATCH `/api/v1/notifications/read-all`** — Mark all as read
   - Call `markAllRead(tenantId, userId)`
   - Return `{ success: true }`

**Update `apps/api/src/services/ticket.service.ts`** to call notification service after ticket events:

1. **In `createTicket()`** — after ticket created and TicketActivity logged:
   ```typescript
   import { notifyTicketCreated } from './notification.service.js';
   // ... after ticket create
   void notifyTicketCreated(tenantId, createdTicket, actorId);
   ```
   Use `void` prefix to fire-and-forget (don't await — notification failure should not block ticket creation).

2. **In `updateTicket()`** — after status changes:
   - If status changed to RESOLVED: `void notifyTicketResolved(tenantId, updatedTicket, actorId);`
   - If assignedToId changed: `void notifyTicketAssigned(tenantId, updatedTicket, newAssignedToId, actorId);`
   - For other field changes: `void notifyTicketUpdated(tenantId, updatedTicket, changedFields, actorId);`

3. **In `addComment()`** — after comment created:
   ```typescript
   import { notifyTicketCommented } from './notification.service.js';
   // ... after comment create, fetch ticket for context
   void notifyTicketCommented(tenantId, ticket, createdComment, actorId);
   ```

4. **In `assignTicket()`** — after assignment:
   ```typescript
   void notifyTicketAssigned(tenantId, updatedTicket, assignedToId, actorId);
   ```

All notification calls are fire-and-forget (void prefix, no await). Wrap each in try/catch that logs errors but never throws — a notification failure must not prevent the ticket operation from completing.
