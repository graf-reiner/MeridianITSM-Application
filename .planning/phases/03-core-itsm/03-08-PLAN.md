2. **Create `apps/api/src/services/report.service.ts`**:

   Import `prisma` from `@meridian/db`, `stringify` from `csv-stringify/sync`.

   **Export `async function getDashboardStats(tenantId: string)`**:
   - Total tickets: `prisma.ticket.count({ where: { tenantId } })`
   - Open tickets: count where status IN ('NEW', 'OPEN', 'IN_PROGRESS', 'PENDING')
   - Resolved today: count where status = 'RESOLVED' AND resolvedAt >= start of today
   - Overdue (SLA breached): count where slaBreachAt < now AND status NOT IN ('RESOLVED', 'CLOSED', 'CANCELLED')
   - Ticket volume by day (last 30 days): `prisma.$queryRaw` grouping by `DATE(createdAt)` for chart
   - Ticket volume by priority: count grouped by priority
   - Top categories: count grouped by categoryId (top 5)
   - Recent activity: last 10 TicketActivity records across all tickets
   - Return all as `{ totalTickets, openTickets, resolvedToday, overdueTickets, volumeByDay, volumeByPriority, topCategories, recentActivity }`

   **Export `async function getTicketReport(tenantId: string, filters: { dateFrom?, dateTo?, status?, priority?, assignedToId?, categoryId?, format: 'csv'|'json' })`**:
   - Query tickets with filters, include assignedTo (firstName, lastName), category (name), queue (name)
   - Cap at 5000 records for memory protection
   - If format='csv': use `stringify()` with header: `['Ticket Number', 'Title', 'Status', 'Priority', 'Assignee', 'Category', 'Created', 'Resolved']`
   - Map each ticket: `TKT-${String(t.ticketNumber).padStart(5, '0')}`, title, status, priority, `${assignedTo.firstName} ${assignedTo.lastName}` or '', category name, createdAt ISO, resolvedAt ISO or ''
   - If format='json': return raw ticket array
   - Return `{ data: string|object[], format, count }`

   **Export `async function getSlaComplianceReport(tenantId: string, filters: { dateFrom?, dateTo? })`**:
   - Query resolved/closed tickets in date range that had an SLA assigned
   - For each: breached = (resolvedAt > slaBreachAt)
   - Calculate: totalWithSla, breachedCount, complianceRate = ((total - breached) / total * 100)
   - Average response time: avg(firstResponseAt - createdAt) in minutes for tickets with firstResponseAt
   - Average resolution time: avg(resolvedAt - createdAt) in minutes for resolved tickets
   - Breakdown by priority: for each P1-P4, count total + breached + avg resolution time
   - Return `{ totalWithSla, breachedCount, complianceRate, avgResponseMinutes, avgResolutionMinutes, byPriority }`

   **Export `async function getChangeReport(tenantId: string, filters)`** (REPT-03):
   - Query Change model with date filters
   - Count by status, by risk level, by type
   - Note: Change model exists in schema but CRUD is Phase 4. This report infrastructure works once changes exist.
   - Return `{ totalChanges, byStatus, byRiskLevel, byType }`

   **Export `async function getSystemHealth(tenantId: string)`** (REPT-07):
   - Create temporary Queue instances for each QUEUE_NAMES entry
   - Call `queue.getJobCounts('active', 'waiting', 'completed', 'failed', 'delayed')` for each
   - Close queue instances after reading
   - Get DB stats: user count, ticket count, article count for tenant
   - Return `{ queues: { name, active, waiting, completed, failed, delayed }[], dbStats: { users, tickets, articles } }`
