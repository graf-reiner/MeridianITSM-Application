    2. Replace apps/api/src/plugins/plan-gate.ts entirely:
       - IMPORTANT: The current planGatePreHandler is registered globally on ALL protected routes. The new implementation must remain a single function export BUT support per-route resource specification. Design:
         - The global planGatePreHandler checks ONLY subscription status (ACTIVE/TRIALING). It does NOT check numeric limits (that would require knowing which resource).
         - Export a NEW function `planGate(resource: PlanResource, currentCountFn?: (tenantId: string) => Promise<number>)` that returns a preHandler for route-level use.
         - The global planGatePreHandler calls the status-check portion only.
       - Implementation of planGate(resource, countFn):
         a. Extract tenantId from request.user (type assertion: { tenantId: string })
         b. Check Redis cache key `plan:${tenantId}` — if hit, parse JSON
         c. If cache miss: query prisma.tenantSubscription.findUnique({ where: { tenantId }, include: { plan: true } })
         d. If no subscription: return 402 { error: 'NO_SUBSCRIPTION' }
         e. Cache result in Redis with SETEX 60s TTL
         f. Check status: if not in ['ACTIVE', 'TRIALING'], return 402 { error: 'SUBSCRIPTION_INACTIVE', status }
         g. For feature resources: check if resource is in plan.limitsJson.features[]. If not: return 402 { error: 'PLAN_LIMIT_EXCEEDED', feature: resource, upgradeTier }
         h. For numeric resources with countFn: get limit from limitsJson[getLimitKey(resource)]. If limit !== -1 (unlimited) and current >= limit: return 402 { error: 'PLAN_LIMIT_EXCEEDED', limit, current, feature: resource, upgradeTier }
         i. upgradeTier: STARTER->'PROFESSIONAL', PROFESSIONAL->'BUSINESS', BUSINESS->'ENTERPRISE', ENTERPRISE->'ENTERPRISE'
       - The response format MUST match CONTEXT.md exactly: `{ error: 'PLAN_LIMIT_EXCEEDED', limit: 5, current: 5, feature: 'agents', upgradeTier: 'PROFESSIONAL' }`
