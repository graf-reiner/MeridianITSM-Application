    1. Create apps/owner/src/app/api/tenants/[id]/lifecycle/route.ts:
       - POST handler with body: { action: 'suspend' | 'unsuspend' | 'delete' | 'extend_trial' | 'apply_grace_period', params?: { days?: number } }
       - suspend: set Tenant.status = 'SUSPENDED', Tenant.suspendedAt = now, TenantSubscription.status = 'SUSPENDED', invalidate Redis plan cache
       - unsuspend: set Tenant.status = 'ACTIVE', Tenant.suspendedAt = null, TenantSubscription.status = 'ACTIVE'
       - delete: soft delete — set Tenant.status = 'DELETED', Tenant.deletedAt = now (30-day recovery window)
       - extend_trial: update TenantSubscription.trialEnd += params.days (default 7), ensure status remains TRIALING
       - apply_grace_period: extend currentPeriodEnd by 3 days, set status to ACTIVE if PAST_DUE
       - All actions invalidate Redis plan cache: fetch from apps/api Redis instance or use local Redis connection

    2. Create apps/owner/src/app/api/tenants/[id]/impersonate/route.ts:
       - POST handler
       - Extract ownerUserId and email from verified owner JWT
       - Call generateImpersonationToken(ownerUserId, tenantId, ownerEmail)
       - Return { impersonationToken, expiresAt: 15 minutes from now, tenantSlug }
       - The frontend uses this token to construct a URL: https://{slug}.domain.com?impersonation_token={token}

    3. Create apps/owner/src/app/api/tenants/[id]/notes/route.ts:
       - GET: list OwnerNote records for tenant, ordered by createdAt desc
       - POST: create new OwnerNote with { content, isPrivate? }
       - Body validation with inline schema

    4. Create apps/owner/src/app/api/dashboard/route.ts:
       - GET handler returning dashboard metrics:
         - totalTenants: count Tenant where status != DELETED
         - activeTenants: count where status = ACTIVE
         - trialingTenants: count TenantSubscription where status = TRIALING
         - suspendedTenants: count where status = SUSPENDED
         - mrr: sum SubscriptionPlan.monthlyPriceUsd for ACTIVE subscriptions (join)
         - arr: mrr * 12
         - recentActivity: last 10 tenants created
         - mrrHistory: monthly MRR aggregation for last 12 months (SQL aggregation)
  </action>
