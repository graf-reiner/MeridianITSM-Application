Create apps/api/src/services/change.service.ts:

1. ALLOWED_TRANSITIONS constant:
   NEW: ['ASSESSMENT', 'CANCELLED']
   ASSESSMENT: ['APPROVAL_PENDING', 'CANCELLED']
   APPROVAL_PENDING: ['APPROVED', 'REJECTED', 'CANCELLED']
   APPROVED: ['SCHEDULED', 'CANCELLED']
   REJECTED: [] (terminal)
   SCHEDULED: ['IMPLEMENTING', 'CANCELLED']
   IMPLEMENTING: ['REVIEW']
   REVIEW: ['COMPLETED', 'IMPLEMENTING'] (reopen for rework)
   COMPLETED: [] (terminal)
   CANCELLED: [] (terminal)

2. getInitialStatus(type): STANDARD -> 'APPROVED', EMERGENCY -> 'APPROVAL_PENDING', NORMAL -> 'NEW'

3. createChange(prisma, tenantId, data, userId):
   - $transaction with FOR UPDATE lock for sequential changeNumber:
     SELECT COALESCE(MAX("changeNumber"), 0) + 1 AS next FROM changes WHERE "tenantId" = ${tenantId}::uuid FOR UPDATE
   - Set initial status via getInitialStatus(data.type)
   - Create Change with: title, description, type, implementationPlan, backoutPlan, testingPlan, riskLevel (or calculate via calculateRiskScore), requestedById=userId, assignedToId, scheduledStart, scheduledEnd
   - Log ChangeActivity: { activityType: 'CREATED', actorId: userId }
   - For STANDARD: also log 'STATUS_CHANGED' from NEW to APPROVED (auto-approved)
   - For NORMAL: if approvers provided in data.approvers[], create ChangeApproval records with sequenceOrder 0,1,2...
   - For EMERGENCY: if approvers provided, create ChangeApproval records, fire CHANGE_APPROVAL notifications immediately
   - Fire-and-forget notification to assignee if set (CHANGE_UPDATED type)

4. getChange(prisma, tenantId, changeId):
   - Include approvals (with approver user name/email, ordered by sequenceOrder), activities (ordered by createdAt DESC), changeAssets (with asset details), changeApplications (with app details), cabMeetingChanges (with meeting title/date)

5. listChanges(prisma, tenantId, filters):
   - Filters: status?, type?, riskLevel?, assignedToId?, requestedById?, search? (title, changeNumber), dateRange? (scheduledStart between)
   - Pagination: page, pageSize
   - Also support: calendarRange? (scheduledStart/End overlap with date range) for calendar view data (CHNG-09)

6. transitionStatus(prisma, tenantId, changeId, newStatus, userId):
   - Fetch current change, validate newStatus is in ALLOWED_TRANSITIONS[currentStatus]
   - If invalid: throw Error with message "Cannot transition from {current} to {newStatus}" (caller returns 409)
   - If transitioning to APPROVAL_PENDING: verify at least one approver exists
   - If transitioning to APPROVED: verify all approvers have status APPROVED (for NORMAL type)
   - Update status, log ChangeActivity with oldValue/newValue
   - Fire notification (CHANGE_UPDATED) to requestedBy and assignedTo

7. addApprover(prisma, tenantId, changeId, approverId, sequenceOrder):
   - Create ChangeApproval with status PENDING
   - Fire notification (CHANGE_APPROVAL) to the approver
   - Log ChangeActivity

8. recordApproval(prisma, tenantId, changeId, approverId, decision, comments?):
   - decision: 'APPROVED' or 'REJECTED'
   - Find ChangeApproval by changeId + approverId
   - Verify it is the current approver's turn (sequenceOrder = min of PENDING approvals)
   - Update approval status, decidedAt = now, comments
   - If REJECTED: transition change to REJECTED status
   - If APPROVED and all approvers approved: transition change to APPROVED status
   - If APPROVED but more approvers pending: notify next approver
   - Log ChangeActivity

9. getCollisions(prisma, tenantId, scheduledStart, scheduledEnd, excludeChangeId?):
   - Query changes where status IN ('SCHEDULED', 'IMPLEMENTING') AND tenantId matches AND scheduledStart < end AND scheduledEnd > start (overlap check) AND id != excludeChangeId
   - Return colliding changes

10. calculateRiskScore(type, affectedCICount?, affectedAppCriticality?):
    - Base score: EMERGENCY=3, NORMAL=1, STANDARD=0
    - +1 per affected CI (capped at 3)
    - +1 if any affected app is CRITICAL criticality
    - Map total to RiskLevel: 0-1=LOW, 2-3=MEDIUM, 4-5=HIGH, 6+=CRITICAL
    - Return suggested RiskLevel

11. linkAsset(prisma, tenantId, changeId, assetId):
    - Create ChangeAsset junction record

12. linkApplication(prisma, tenantId, changeId, applicationId):
    - Create ChangeApplication junction record
