Create apps/api/src/services/asset.service.ts:

1. ASSET_TRANSITIONS map (status lifecycle guard):
   - IN_STOCK: ['DEPLOYED', 'IN_REPAIR', 'DISPOSED']
   - DEPLOYED: ['IN_REPAIR', 'RETIRED']
   - IN_REPAIR: ['DEPLOYED', 'RETIRED']
   - RETIRED: ['DISPOSED']
   - DISPOSED: [] (terminal)

2. createAsset(prisma, tenantId, data, actorId):
   - Use $transaction with $queryRaw FOR UPDATE lock on assets table for sequential assetTag:
     SELECT COALESCE(MAX(CAST(SUBSTRING("assetTag" FROM 'AST-([0-9]+)') AS INTEGER)), 0) + 1 AS next FROM assets WHERE "tenantId" = ${tenantId}::uuid FOR UPDATE
   - Format: AST-{padded 5 digits} e.g. AST-00001
   - Create asset with provided fields: serialNumber, manufacturer, model, status (default IN_STOCK), purchaseDate, purchaseCost, warrantyExpiry, assignedToId, siteId, hostname, operatingSystem, osVersion, cpuModel, cpuCores, ramGb, disks, networkInterfaces, softwareInventory, customFields

3. getAsset(prisma, tenantId, assetId): Find by id + tenantId, include site relation

4. listAssets(prisma, tenantId, filters):
   - Filters: status?, assignedToId?, siteId?, search? (matches assetTag, serialNumber, hostname, manufacturer, model)
   - Pagination: page (default 1), pageSize (default 25)
   - Return { data: Asset[], total: number, page: number, pageSize: number }

5. updateAsset(prisma, tenantId, assetId, data, actorId):
   - If status change: validate against ASSET_TRANSITIONS. If invalid, throw Error('Invalid status transition from X to Y')
   - Update allowed fields

6. deleteAsset(prisma, tenantId, assetId): Soft-delete by setting status to DISPOSED, or hard delete if no references

Create apps/api/src/routes/v1/assets/index.ts:
- POST /api/v1/assets — requirePermission('assets.create'), body: { serialNumber?, manufacturer?, model?, status?, purchaseDate?, purchaseCost?, warrantyExpiry?, assignedToId?, siteId? }
- GET /api/v1/assets — requirePermission('assets.read'), query: { status?, assignedToId?, siteId?, search?, page?, pageSize? }
- GET /api/v1/assets/:id — requirePermission('assets.read')
- PUT /api/v1/assets/:id — requirePermission('assets.update')
- DELETE /api/v1/assets/:id — requirePermission('assets.delete')

For PRTL-05 (portal endpoint): GET /api/v1/assets with assignedToId=userId from JWT — no special route needed, the list endpoint filters by assignedToId.

Register in apps/api/src/routes/v1/index.ts:
