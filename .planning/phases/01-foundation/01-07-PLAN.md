# Plan 01-07: Apply AUTH_RATE_LIMIT to Auth Routes (Gap Closure)

<objective>
Import AUTH_RATE_LIMIT from the rate-limit plugin and apply it to the login and password-reset route handlers so auth endpoints enforce the stricter 5 requests per 15 minutes limit instead of the global 100/min default.
</objective>

<context>
**Gap from VERIFICATION.md:** AUTH_RATE_LIMIT constant (max: 5, timeWindow: '15 minutes') is exported from `apps/api/src/plugins/rate-limit.ts` but is never imported or applied to the login or password-reset routes. Auth endpoints currently fall through to the global default (100 req/min).
</context>

<tasks>

<task id="1" name="Apply AUTH_RATE_LIMIT to login and password-reset routes">
<files>
- apps/api/src/routes/auth/login.ts
- apps/api/src/routes/auth/password-reset.ts
</files>

<read_first>
- apps/api/src/routes/auth/login.ts
- apps/api/src/routes/auth/password-reset.ts
- apps/api/src/plugins/rate-limit.ts
</read_first>

<action>
1. In `apps/api/src/routes/auth/login.ts`:
   - Add import: `import { AUTH_RATE_LIMIT } from '../plugins/rate-limit.js';`
   - Add route config to the POST handler: `{ config: { rateLimit: AUTH_RATE_LIMIT } }` as the options parameter in `app.post('/api/auth/login', { config: { rateLimit: AUTH_RATE_LIMIT } }, async (request, reply) => { ... })`

2. In `apps/api/src/routes/auth/password-reset.ts`:
   - Add import: `import { AUTH_RATE_LIMIT } from '../plugins/rate-limit.js';`
   - Apply `{ config: { rateLimit: AUTH_RATE_LIMIT } }` to both POST route handlers (request-reset and reset-password)
</action>

<verify>
<automated>grep -c "AUTH_RATE_LIMIT" apps/api/src/routes/auth/login.ts && grep -c "AUTH_RATE_LIMIT" apps/api/src/routes/auth/password-reset.ts</automated>
</verify>
