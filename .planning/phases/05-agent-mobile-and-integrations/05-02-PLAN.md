1. **Webhook delivery worker** — Create `apps/worker/src/workers/webhook-delivery.ts`:

   Interface:
   ```typescript
   export interface WebhookDeliveryJobData {
     tenantId: string;
     webhookId: string;
     event: string;
     payload: Record<string, unknown>;
   }
   ```

   Worker logic:
   - Find webhook by webhookId + tenantId where isActive=true. If not found, return silently.
   - Build JSON body: `{ event, data: payload, timestamp: new Date().toISOString() }`
   - If webhook.secret exists: compute HMAC-SHA256 signature: `createHmac('sha256', webhook.secret).update(body).digest('hex')`
   - Set headers: `Content-Type: application/json`, `X-Meridian-Event: {event}`, and if signature: `X-Meridian-Signature: sha256={signature}`
   - Merge any custom webhook.headers (Json field)
   - POST to webhook.url with fetch(), capture startMs for timing
   - Record WebhookDelivery: `{ tenantId, webhookId, event, payload, responseStatus, responseBody (first 1000 chars), attemptCount: (job.attemptsMade ?? 0) + 1, success: (status >= 200 && < 300), deliveredAt: success ? new Date() : null }`
   - On success: reset consecutiveFailures to 0 on the Webhook record
   - On failure: increment consecutiveFailures. If consecutiveFailures >= 50, set isActive=false on webhook
   - If not success: throw error to trigger BullMQ retry

   Custom backoff strategy (register in Worker constructor options):
   ```typescript
   const BACKOFF_DELAYS = [60_000, 300_000, 1_800_000, 7_200_000, 43_200_000]; // 1m, 5m, 30m, 2h, 12h
   // In Worker options:
   settings: {
     backoffStrategy: (attemptsMade: number) => BACKOFF_DELAYS[attemptsMade - 1] ?? 43_200_000,
   }
   ```

   Worker options: `{ connection: bullmqConnection, concurrency: 5 }`
