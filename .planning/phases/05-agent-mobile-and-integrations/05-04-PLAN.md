2. **WmiCollector** (Windows) — `apps/inventory-agent/src/InvAgent.Collectors/Windows/WmiCollector.cs`:
   - Add NuGet: `System.Management` (only on Windows — use conditional ItemGroup in .csproj: `<PackageReference Condition="'$(OS)' == 'Windows_NT'" ...>`)
   - Implements ICollector
   - Uses `ManagementObjectSearcher` for WMI queries:
     - `SELECT * FROM Win32_OperatingSystem` → OsInfo
     - `SELECT * FROM Win32_ComputerSystem` → HardwareInfo (Manufacturer, Model)
     - `SELECT * FROM Win32_BIOS` → SerialNumber
     - `SELECT * FROM Win32_Processor` → CpuInfo[]
     - `SELECT * FROM Win32_PhysicalMemory` → TotalMemoryBytes (sum Capacity)
     - `SELECT * FROM Win32_DiskDrive` → DiskInfo[]
     - `SELECT * FROM Win32_NetworkAdapterConfiguration WHERE IPEnabled=True` → NetworkInterface[]
     - `SELECT * FROM Win32_Product` → SoftwareEntry[] (fallback: registry uninstall key)
     - `SELECT * FROM Win32_Service` → ServiceEntry[]
     - `SELECT * FROM Win32_Process` → ProcessEntry[]
     - `SELECT * FROM Win32_UserAccount WHERE LocalAccount=True` → LocalUser[]
   - All WMI calls wrapped in try/catch — individual collection failures logged but don't abort entire collection

3. **ProcCollector** (Linux) — `apps/inventory-agent/src/InvAgent.Collectors/Linux/ProcCollector.cs`:
   - Implements ICollector
   - OS info: parse `/etc/os-release` for NAME, VERSION_ID; `uname -m` for architecture
   - Hardware: `/sys/class/dmi/id/sys_vendor`, `/sys/class/dmi/id/product_name`, `/sys/class/dmi/id/product_serial`
   - CPU: parse `/proc/cpuinfo` for model name, cores; `/sys/devices/system/cpu/` for thread count
   - Memory: parse `/proc/meminfo` for MemTotal
   - Disks: parse `/proc/partitions` or `lsblk --json`
   - Network: parse `/sys/class/net/*/address` and run `ip addr`
   - Software: shell out to `dpkg-query -W -f '${Package}\t${Version}\t${Maintainer}\n'` (Debian) OR `rpm -qa --qf '%{NAME}\t%{VERSION}\t%{VENDOR}\n'` (RHEL)
   - Services: parse `systemctl list-units --type=service --all --no-legend`
   - Processes: parse `/proc/[pid]/stat` and `/proc/[pid]/status` for PID, name, CPU, memory
   - Local users: parse `/etc/passwd`, check admin via groups (wheel/sudo)

4. **MacOsCollector** — `apps/inventory-agent/src/InvAgent.Collectors/MacOs/MacOsCollector.cs`:
   - Implements ICollector
   - OS: run `sw_vers` for ProductName, ProductVersion, BuildVersion
   - Hardware: run `system_profiler SPHardwareDataType -json` — parse for Model, SerialNumber, CPU, Memory
   - Disks: run `system_profiler SPStorageDataType -json`
   - Network: run `system_profiler SPNetworkDataType -json`
   - Software: run `system_profiler SPApplicationsDataType -json`
   - Services: run `launchctl list` for launchd services
   - Processes: run `ps aux` and parse
   - Local users: run `dscl . -list /Users` and `dscacheutil -q group -a name admin`
   - All commands run via `Process.Start` with redirected stdout

5. **PrivacyFilter** — Create `apps/inventory-agent/src/InvAgent.Privacy/PrivacyFilter.cs`:
   ```csharp
   namespace InvAgent.Privacy;
   using InvAgent.Models;

   public static class PrivacyFilter
   {
       public static InventoryPayload Apply(InventoryPayload payload, string tier)
       {
           return tier switch
           {
               "full" => payload,  // No filtering
               "restricted" => ApplyRestricted(payload),
               "anonymized" => ApplyAnonymized(payload),
               _ => payload,
           };
       }
   }
   ```
   - `restricted`: Remove LocalUsers entirely, remove Software publisher, remove Process names (keep PIDs), strip serial numbers
   - `anonymized`: All of restricted, plus hash hostnames, MAC addresses, and IP addresses with SHA-256 (first 12 chars)
