    1. Create apps/owner/src/lib/stripe-admin.ts:
       - Import Stripe from 'stripe'
       - Create stripe instance: new Stripe(process.env.STRIPE_SECRET_KEY!)
       - Export retryInvoice(invoiceId: string): calls stripe.invoices.pay(invoiceId)
       - Export listTenantInvoices(stripeCustomerId: string): calls stripe.invoices.list({ customer, limit: 50 })
       - Export getStripeRevenue(): calls stripe.balanceTransactions.list with date range, sums amounts
       - All wrapped in try/catch with typed error handling

    2. Create apps/owner/src/app/api/billing/route.ts:
       - GET: return billing overview for all tenants
       - Query TenantSubscription with include: { tenant: true, plan: true }
       - Group by status (active, trialing, past_due, canceled, suspended)
       - Calculate total MRR from active + trialing subscriptions
       - Return { overview: { totalMrr, byStatus }, tenants: [...] }

    3. Create apps/owner/src/app/api/billing/[tenantId]/retry/route.ts:
       - POST handler
       - Get TenantSubscription for tenantId, get stripeCustomerId
       - Fetch latest unpaid invoice: stripe.invoices.list({ customer, status: 'open', limit: 1 })
       - Call retryInvoice(invoice.id)
       - Return { success: true, invoiceId, newStatus }

    4. Create apps/owner/src/app/api/plans/route.ts:
       - GET: list all SubscriptionPlan records with counts of active subscriptions per plan
       - Return { plans: [{ ...plan, activeCount }] }

    5. Create apps/owner/src/app/api/plans/[id]/route.ts:
       - GET: single plan detail
       - PUT: update plan fields (displayName, monthlyPriceUsd, annualPriceUsd, limitsJson, stripePriceIdMonthly, stripePriceIdAnnual, isPublic)
       - Validate body: limitsJson must have maxUsers (number), maxAgents (number), maxSites (number), features (string[])
       - Do NOT allow changing the name (tier) — it's the unique identifier

    6. Create apps/owner/src/app/api/system/route.ts:
       - GET: return system health info
       - For each queue in QUEUE_NAMES: get job counts using BullMQ Queue API (create Queue instances connected to Redis, call getJobCounts())
       - Return { queues: [{ name, waiting, active, completed, failed, delayed }], redisStatus: 'connected' | 'disconnected' }
       - For maintenance broadcast: POST with { message, expiresInMinutes? } — stores in Redis key 'maintenance:broadcast' with TTL

    7. Create apps/owner/src/app/api/audit/route.ts:
       - GET: cross-tenant audit log viewer
       - Query params: page, limit, tenantId? (optional filter), action?, resource?, startDate?, endDate?
       - IMPORTANT: This is the ONLY endpoint that queries AuditLog WITHOUT tenantId filter — it's owner-only capability
       - Include tenant name and user email via relations
       - Return { logs: [...], total, page, pageCount }
