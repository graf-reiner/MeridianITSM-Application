1. **Agent external routes** — Create `apps/api/src/routes/v1/agents/index.ts` exporting `agentRoutes(app: FastifyInstance)`:

   **POST /api/v1/agents/enroll** (public within external scope — move to external routes):
   - Accept body: `{ token: string, hostname: string, platform: string, agentVersion: string }`
   - Hash token with `createHash('sha256').update(token).digest('hex')`
   - Find AgentEnrollmentToken by tokenHash where isActive=true AND (expiresAt is null OR expiresAt > now)
   - If not found: 401 `{ error: 'Invalid or expired enrollment token' }`
   - If enrollCount >= maxEnrollments: 409 `{ error: 'Enrollment token max usage reached' }`
   - Generate agentKey: `randomBytes(32).toString('hex')`
   - In $transaction: create Agent record (tenantId from token, agentKey, hostname, platform cast, agentVersion, status='ACTIVE', enrolledAt=new Date()) AND increment enrollCount on token
   - Return 201 `{ agentKey, agentId }`

   **POST /api/v1/agents/heartbeat** (external scope — agent authenticates with agentKey via custom header):
   - Accept `Authorization: AgentKey <key>` header. Look up agent by agentKey where status='ACTIVE'
   - Update agent: `lastHeartbeatAt = new Date(), status = 'ACTIVE'`
   - Accept optional body: `{ agentVersion?: string, metrics?: Record<string, unknown> }`
   - If metrics provided, create MetricSample record
   - Return 200 `{ ok: true }`

   **POST /api/v1/agents/inventory** (external scope):
   - Same AgentKey auth as heartbeat
   - Accept body with inventory payload (os, hostname, hardware, software, services, processes, network, localUsers)
   - Create InventorySnapshot record: `{ tenantId: agent.tenantId, agentId: agent.id, data: body as Json, collectedAt: new Date() }`
   - Return 201 `{ snapshotId }`

   **POST /api/v1/agents/cmdb-sync** (external scope):
   - Same AgentKey auth as heartbeat
   - Accept body with CI payload data
   - Enqueue cmdb-reconciliation job: `cmdbReconciliationQueue.add('agent-sync', { tenantId: agent.tenantId, agentId: agent.id, payload: body })`
   - Return 202 `{ status: 'queued' }`

   **Agent auth helper** — Create a shared `resolveAgent(request, reply)` function that:
   - Reads `Authorization: AgentKey <key>` header
   - Looks up agent by agentKey where status != 'DEREGISTERED'
   - Returns the agent record or replies 401

   Register agent enrollment in the external routes scope (apps/api/src/routes/external/index.ts — update to add enroll route).
   Register heartbeat/inventory/cmdb-sync also in external scope (they use AgentKey, not JWT).

2. **Admin agent management routes** — Create `apps/api/src/routes/v1/settings/agents.ts` exporting `agentSettingsRoutes(app: FastifyInstance)`:

   These are tenant-admin routes (JWT protected, admin only via hasPermission 'settings.update') used by the web settings UI (Plan 09).

   **GET /api/v1/settings/agents** (JWT, admin):
   - List all agents for tenant: `prisma.agent.findMany({ where: { tenantId }, orderBy: { lastHeartbeatAt: 'desc' } })`
   - Return `{ id, hostname, platform, status, lastHeartbeatAt, agentVersion, enrolledAt }`
   - Compute display status: if lastHeartbeatAt > 24h ago and status is ACTIVE, show as 'STALE'

   **GET /api/v1/settings/agents/tokens** (JWT, admin):
   - List enrollment tokens for tenant: `prisma.agentEnrollmentToken.findMany({ where: { tenantId } })`
   - Return `{ id, prefix (first 8 chars of tokenHash), enrollCount, maxEnrollments, expiresAt, isActive, createdAt }`

   **POST /api/v1/settings/agents/tokens** (JWT, admin):
   - Accept body: `{ maxEnrollments?: number, expiresAt?: string }`
   - Generate raw token: `randomBytes(32).toString('hex')`
   - Hash token: `createHash('sha256').update(rawToken).digest('hex')`
   - Create AgentEnrollmentToken: `{ tenantId, tokenHash, maxEnrollments: maxEnrollments ?? null, expiresAt: expiresAt ? new Date(expiresAt) : null, isActive: true, enrollCount: 0 }`
   - Return 201 `{ id, token: rawToken, expiresAt, maxEnrollments }` — token only returned once (like API key pattern)

   **DELETE /api/v1/settings/agents/tokens/:id** (JWT, admin):
   - Set isActive=false on AgentEnrollmentToken where id and tenantId match
   - Return 200 `{ ok: true }`

   **DELETE /api/v1/settings/agents/:id** (JWT, admin):
   - Delete Agent record where id and tenantId match (also cascades InventorySnapshot records)
   - Return 200 `{ ok: true }`

   Register agentSettingsRoutes in the settings routes index (`apps/api/src/routes/v1/settings/index.ts`):

3. **Push registration routes** — Create `apps/api/src/routes/v1/push/index.ts` exporting `pushRoutes(app: FastifyInstance)`:

   **POST /api/v1/push/register** (JWT protected):
   - Accept body: `{ token: string, platform: 'IOS' | 'ANDROID', deviceId: string }`
   - Upsert DeviceToken by `{ userId_deviceId: { userId, deviceId } }` where userId from JWT, tenantId from JWT
   - Set `token, platform, isActive: true, updatedAt: new Date()`
   - Return 200 `{ ok: true }`

   **DELETE /api/v1/push/unregister** (JWT protected):
   - Accept body: `{ deviceId: string }`
   - Update DeviceToken where userId+deviceId: set `isActive: false`
   - Return 200 `{ ok: true }`

   Register pushRoutes in v1Routes (apps/api/src/routes/v1/index.ts).

4. **API key management routes** — Create `apps/api/src/routes/v1/settings/api-keys.ts` exporting `apiKeySettingsRoutes(app: FastifyInstance)`:

   **POST /api/v1/settings/api-keys** (JWT, admin only via hasPermission check for 'settings.update'):
   - Accept body: `{ name: string, scopes: string[], expiresAt?: string, rateLimit?: number }`
   - Generate raw key: `randomBytes(32).toString('hex')`
   - Create prefix from first 8 chars: `rawKey.slice(0, 8)`
   - Hash full key: `createHash('sha256').update(rawKey).digest('hex')`
   - Store in ApiKey: `{ tenantId, name, keyHash, prefix, scopes, expiresAt, rateLimit: rateLimit ?? 100, isActive: true, createdById: userId }`
   - Return 201 `{ id, name, key: rawKey, prefix, scopes, createdAt }` — key only returned once

   **GET /api/v1/settings/api-keys** (JWT, admin):
   - List all API keys for tenant: `{ id, name, prefix, scopes, rateLimit, lastUsedAt, createdAt, isActive }`
   - Never return keyHash or full key

   **DELETE /api/v1/settings/api-keys/:id** (JWT, admin):
   - Set isActive=false on ApiKey where id and tenantId match
   - Return 200 `{ ok: true }`

   Register apiKeySettingsRoutes in the settings routes index (apps/api/src/routes/v1/settings/index.ts).
