## Scope of This Phase

Phase 5 has three distinct surface areas, each with its own UI scope:

1. **Web app additions** — new settings pages in `apps/web` (Agent management, API Keys, Webhooks, Alert Channels) plus QR code generation for mobile onboarding. Follows existing dashboard page patterns exactly.
2. **Mobile app** — full React Native + Expo scaffold in `apps/mobile`. Native platform UI conventions (iOS/Android) with tenant branding applied to accent color and logo.
3. **Agent local web UI** — ASP.NET Core static SPA at `127.0.0.1:8787` serving diagnostic views. Minimal, functional, loopback-only. Separate design context from the main web app.

## Interaction Contracts

### Web App

**Agent status badge:** Rendered as an inline pill (`border-radius: 9999px`, `padding: 2px 8px`, `font-size: 12px`, `font-weight: 500`) using the status colors from the Color section. No interactive behavior — display only.

**API key creation flow:** Modal opens on "Create API Key" click. Form fields: name (required), scopes (multi-checkbox: tickets.read, tickets.write, assets.read, ci.read), optional rate limit override. On submit: POST `/api/v1/settings/api-keys`. Response shows the full key in a highlighted read-only input with a "Copy" button. Key is NOT shown after modal close. Banner persists at top of list: "API key created. Copy it now — it will not be shown again."

**Webhook test delivery:** "Send Test" button triggers POST `/api/v1/webhooks/:id/test`. Button enters loading state (`cursor: not-allowed`, reduced opacity) while the request is in flight. Success: green inline toast "Test delivery queued." Failure: red inline toast "Test failed — check endpoint URL."

**Webhook delivery history timeline:** Per-webhook detail page shows a timeline list. Each entry: status badge (success/failed/pending), HTTP status code, response time in ms, ISO timestamp, retry count badge (if > 0), expandable payload preview (collapsible `<pre>` block with monospace 12px font). Last 30 days retained per CONTEXT.md.

**Enrollment token generation:** Admin sets optional expiry (date picker) and max-enrollment count (number input, default blank = unlimited). Generated token displayed in read-only input with copy button. Same one-time-display pattern as API key.

**Inline confirmation dialogs:** All destructive confirmations use an inline confirmation state on the row (not a separate modal). The row's action button changes to "Confirm Revoke?" with a red confirm button and a "Cancel" text link adjacent. This matches the expected dense-table interaction pattern of the existing app.

### Mobile App

**Bottom tab bar:** 5 tabs (Dashboard, Tickets, Knowledge, Assets, Profile). Active tab: accent color icon + accent color label. Inactive: `#9ca3af` icon and label. Tab bar background: white with 1px top border `#e5e7eb`. Safe area insets respected via `react-native-safe-area-context`.

**Pull-to-refresh:** Implemented on TicketListScreen, KbListScreen, AssetListScreen, and DashboardScreen. Uses React Native `RefreshControl` with accent color spinner.

**Offline banner:** Persistent banner below tab bar when `NetInfo.isConnected === false`. Background `#fef3c7`, text `#92400e`, 13px. Auto-dismisses when connection restored; "Changes synced" toast shown for 2 seconds.

**Optimistic comment submit:** Comment appears immediately in thread with a "Sending..." opacity-0.6 state. On API success: full opacity. On failure: red error inline with "Retry" tap target.

**Photo picker:** Tapping "Add Photo" presents an ActionSheet with "Take Photo" and "Choose from Library" options. Multiple photos allowed per comment (up to 5). Thumbnails shown as 72px squares in a horizontal scroll row within the comment input area. Each thumbnail has an X button (16px, `#dc2626`) for removal.

**Push notification permission request:** Shown once after first successful login. Native iOS/Android permission dialog. If denied: profile tab shows a non-blocking inline prompt "Enable notifications to get ticket updates" with a "Settings" deep-link.

**QR scan screen:** Full-screen camera view with an overlay square guide frame (64px corner marks, white, 2px stroke) centered in the view. Label below: "Point at the QR code from Settings > Agents in MeridianITSM." Cancel button top-right. Manual entry link below.
