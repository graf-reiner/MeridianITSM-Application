Replace apps/worker/src/workers/cmdb-reconciliation.ts stub with real reconciliation logic:

The worker runs as a global sweep (like SLA monitor) — one repeatable cron job every 15 minutes. It does NOT receive per-agent job data; instead it queries all agents with recent inventory.

1. Import prisma client from worker's db setup (same pattern as sla-monitor.ts)
2. Worker processor function:
   a. Query all InventorySnapshots with createdAt > (now - 30 min) that have not been processed (or use a simple "last processed" marker)
      Actually, simpler approach: Query all agents with status ACTIVE, for each agent get their latest InventorySnapshot
   b. For each agent:
      - Get tenantId from agent record
      - Get latest InventorySnapshot
      - Find existing CmdbConfigurationItem where agentId = agent.id AND tenantId
      - If no CI exists:
        * Create new CI: name=snapshot hostname or agent.hostname, type=infer from OS (Windows/Linux workstation -> WORKSTATION, server hints -> SERVER), status=ACTIVE, agentId=agent.id, tenantId
        * Use sequential ciNumber via raw SQL FOR UPDATE (same pattern, duplicated in worker to avoid cross-app imports)
        * Log CmdbChangeRecord: changeType=CREATED, changedBy=AGENT, agentId
        * Update CI fields from snapshot: attributesJson with hardware data, hostname, operatingSystem, etc.
      - If CI exists:
        * Compare fields: hostname, operatingSystem, osVersion, and attributesJson sub-fields
        * For each changed field: create CmdbChangeRecord with changeType=UPDATED, fieldName, oldValue, newValue, changedBy=AGENT, agentId
        * Update CI with new values, set lastSeenAt = now()
      - All writes in single Prisma $transaction scoped by tenantId
   c. Stale CI marking:
      - Query CIs where agentId IS NOT NULL AND lastSeenAt < (now - 24h) AND status = 'ACTIVE'
      - Update each to status = 'INACTIVE'
      - Log CmdbChangeRecord: changeType=UPDATED, fieldName='status', oldValue='ACTIVE', newValue='INACTIVE', changedBy=AGENT
3. Log summary: "Reconciled {N} agents, created {M} CIs, updated {K} CIs, marked {J} stale"
