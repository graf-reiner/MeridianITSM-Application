1. **Create `apps/api/src/services/knowledge.service.ts`**:

   **Article number generation** (same transactional pattern as ticket numbers):
   ```typescript
   async function getNextArticleNumber(tenantId: string, tx: PrismaTransaction): Promise<number> {
     const result = await tx.$queryRaw<[{ next: number }]>`
       SELECT COALESCE(MAX("articleNumber"), 0) + 1 AS next
       FROM knowledge_articles WHERE "tenantId" = ${tenantId}::uuid FOR UPDATE`;
     return result[0].next;
   }
   ```

   **Status transition map**:
   ```typescript
   const ARTICLE_TRANSITIONS: Record<string, string[]> = {
     DRAFT: ['IN_REVIEW', 'PUBLISHED'],
     IN_REVIEW: ['PUBLISHED', 'DRAFT'],
     PUBLISHED: ['RETIRED', 'DRAFT'],
     RETIRED: ['DRAFT'],
   };
   ```

   **Export `async function createArticle(tenantId, data, authorId)`**:
   - Use $transaction for sequential articleNumber
   - data: `{ title, summary?, content, tags?, visibility? }`
   - Return created article

   **Export `async function updateArticle(tenantId, articleId, data, actorId)`**:
   - If status change: validate against ARTICLE_TRANSITIONS
   - If transitioning to PUBLISHED: set `publishedAt = new Date()` if not already set
   - Update article, return updated

   **Export `async function getArticleList(tenantId, filters)`**:
   - filters: `{ status?, visibility?, search?, tags?, authorId?, page?, pageSize? }`
   - For `search`: `OR: [{ title: { contains: search, mode: 'insensitive' } }, { summary: { contains: search, mode: 'insensitive' } }, { tags: { has: search } }]`
   - Paginate, return `{ data, total }`

   **Export `async function getArticleDetail(tenantId, articleId)`**:
   - Find article by id + tenantId
   - Increment viewCount: `prisma.knowledgeArticle.update({ where: { id: articleId, tenantId }, data: { viewCount: { increment: 1 } } })`
   - Return article (do NOT include viewCount increment in the returned data — use the pre-increment value or re-query)

   **Export `async function voteArticle(tenantId, articleId, helpful: boolean)`**:
   - If helpful: `prisma.knowledgeArticle.update({ data: { helpfulCount: { increment: 1 } } })`
   - If not helpful: `prisma.knowledgeArticle.update({ data: { helpfulCount: { decrement: 1 } } })` (min 0)
   - Return updated helpfulCount

   **Export `async function getPublishedArticles(tenantId, filters)`**:
   - Same as getArticleList but enforces `status: 'PUBLISHED'` and `visibility: 'PUBLIC'`
   - Used by portal (end-user) routes

2. **Create `apps/api/src/routes/v1/knowledge/index.ts`**:

   **GET `/api/v1/knowledge`** — List articles (staff view, all statuses)
   - Query params: status, visibility, search, tags (comma-separated), authorId, page, pageSize
   - Call getArticleList

   **GET `/api/v1/knowledge/published`** — List published articles (portal/public view)
   - Call getPublishedArticles
   - No auth check beyond tenant scope — end users can access

   **POST `/api/v1/knowledge`** — Create article
   - preHandler: `requirePermission('knowledge:write')`
   - Body: `{ title: string, summary?: string, content: string, tags?: string[], visibility?: 'PUBLIC'|'INTERNAL' }`
   - Validate: title minLength 1 maxLength 500, content minLength 1
   - Call createArticle(tenantId, body, userId)
   - Return 201

   **GET `/api/v1/knowledge/:id`** — Get article detail (increments view count)
   - Call getArticleDetail

   **PATCH `/api/v1/knowledge/:id`** — Update article
   - preHandler: `requirePermission('knowledge:write')`
   - Body: subset of `{ title, summary, content, tags, visibility, status }`
   - Call updateArticle

   **DELETE `/api/v1/knowledge/:id`** — Delete article
   - preHandler: `requirePermission('knowledge:write')`
   - Return 204

   **POST `/api/v1/knowledge/:id/vote`** — Vote on article
   - Body: `{ helpful: boolean }`
   - Call voteArticle
   - Return updated helpfulCount
