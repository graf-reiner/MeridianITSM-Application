2. **Create `apps/api/src/services/email.service.ts`** (outbound + templates):

   Import `nodemailer` (createTransport), `prisma` from `@meridian/db`, `decrypt` from `@meridian/core`.

   **Export `async function getSmtpTransport(account: EmailAccount)`**:
   - Decrypt `account.smtpPasswordEnc` using `decrypt()`
   - Return `nodemailer.createTransport({ host: account.smtpHost, port: account.smtpPort ?? 587, secure: account.smtpSecure, auth: { user: account.smtpUser, pass: decryptedPassword } })`

   **Export `async function sendEmail(account: EmailAccount, to: string, subject: string, html: string, options?: { inReplyTo?: string, references?: string[], messageId?: string })`**:
   - Get transport via getSmtpTransport
   - Send with `from: account.emailAddress`, `to`, `subject`, `html`, `messageId` (generate: `<${uuid()}@meridian.local>`), `inReplyTo`, `references`
   - Close transport after send
   - Return sent messageId

   **Export `async function renderTemplate(tenantId: string, templateName: string, variables: Record<string, string>)`**:
   - Fetch EmailTemplate where tenantId + name = templateName (or isDefault=true + name as fallback)
   - If no template found, use a hardcoded default HTML template with basic styling
   - Replace `{{variableName}}` patterns in htmlBody and subject with values from variables map
   - Return `{ subject, html }`

   **Export `async function testSmtpConnection(config: { host, port, user, password, secure }): Promise<{ success: boolean, error?: string }>`**:
   - Create transport, call `transport.verify()`, close
   - Return `{ success: true }` or `{ success: false, error: err.message }`

   **Export `async function testImapConnection(config: { host, port, user, password, secure }): Promise<{ success: boolean, error?: string }>`**:
   - Create ImapFlow client, attempt connect + logout
   - Return success/failure

3. **Create `apps/api/src/services/email-inbound.service.ts`** (IMAP polling):

   Import `ImapFlow` from `imapflow`, `simpleParser` from `mailparser`, `prisma` from `@meridian/db`, `decrypt` from `@meridian/core`, `redis` from `../lib/redis.js`, `createTicket`, `addComment` from `./ticket.service.js`, `uploadFile` from `./storage.service.js`.

   **Export `async function isDuplicate(tenantId: string, messageId: string): Promise<boolean>`**:
   - Redis key: `email:msgids:${tenantId}`
   - Use `redis.sismember(key, messageId)` — returns 1 if exists
   - If not exists: `redis.sadd(key, messageId)` and `redis.expire(key, 90 * 24 * 3600)` (90-day TTL)
   - Return boolean

   **Export `async function findTicketByHeaders(tenantId: string, references?: string[], inReplyTo?: string)`**:
   - If `inReplyTo`, search tickets where tenantId matches and customFields contains `outboundMessageId = inReplyTo`
   - If `references` array, iterate and search similarly
   - Return ticket or null

   **Export `async function findTicketBySubject(tenantId: string, subject: string)`**:
   - Regex: `/TKT-(\d{5})/` on subject
   - If match, find ticket where `tenantId` AND `ticketNumber = parseInt(match[1])`
   - Return ticket or null

   **Export `async function pollMailbox(account: EmailAccount)`**:
   - Decrypt `account.imapPasswordEnc`
   - Create ImapFlow client: `{ host: account.imapHost, port: account.imapPort ?? 993, secure: account.imapSecure, auth: { user: account.imapUser, pass: decryptedPassword }, logger: false }`
   - Connect, lock INBOX
   - Fetch unseen messages: `client.fetch({ seen: false }, { envelope: true, source: true }, { uid: true })`
   - For each message:
     - Parse with `simpleParser(message.source)`
     - Check `isDuplicate(account.tenantId, parsed.messageId)` — skip if duplicate
     - Try threading: `findTicketByHeaders(account.tenantId, parsed.references, parsed.inReplyTo)` then `findTicketBySubject(account.tenantId, parsed.subject ?? '')`
     - If existing ticket found: `addComment(account.tenantId, ticket.id, { content: parsed.text ?? parsed.html ?? '', visibility: 'PUBLIC' }, null)`
     - If no ticket: `createTicket(account.tenantId, { title: parsed.subject ?? 'No Subject', description: parsed.text ?? parsed.html ?? '', type: 'INCIDENT', priority: 'MEDIUM', queueId: account.defaultQueueId, categoryId: account.defaultCategoryId, requestedById: lookupUserByEmail(parsed.from?.value?.[0]?.address) }, null)`
     - Store outbound message reference in ticket customFields for future reply threading
     - Process attachments: for each `parsed.attachments`, upload via `uploadFile(attachment.content, key, attachment.contentType)`, create TicketAttachment record. Respect 25MB total size limit per email.
     - Mark message as Seen: `client.messageFlagsAdd({ uid: message.uid }, ['\\Seen'], { uid: true })`
   - Release lock, logout
   - Update `account.lastPolledAt` in DB
