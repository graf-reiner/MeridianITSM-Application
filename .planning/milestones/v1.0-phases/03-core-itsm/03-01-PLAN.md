3. **Create `apps/api/src/services/ticket.service.ts`**:
   - Import `prisma` from `@meridian/db`
   - Import `redis` from `../lib/redis.js`

   **Status transition map** (enforced on every status change):
   ```
   const ALLOWED_TRANSITIONS: Record<string, string[]> = {
     NEW: ['OPEN', 'IN_PROGRESS', 'CANCELLED'],
     OPEN: ['IN_PROGRESS', 'PENDING', 'RESOLVED', 'CANCELLED'],
     IN_PROGRESS: ['PENDING', 'RESOLVED', 'CANCELLED'],
     PENDING: ['OPEN', 'IN_PROGRESS', 'RESOLVED', 'CANCELLED'],
     RESOLVED: ['CLOSED', 'OPEN'],
     CLOSED: [],
     CANCELLED: [],
   };
   ```

   **Export `async function createTicket(tenantId, data, actorId)`**:
   - Use `prisma.$transaction` with `SELECT COALESCE(MAX("ticketNumber"), 0) + 1 AS next FROM tickets WHERE "tenantId" = $1::uuid FOR UPDATE` to get next ticket number
   - Create ticket with all fields from data (title, description, type, priority, categoryId, queueId, assignedToId, requestedById, slaId, tags)
   - If `queueId` provided and queue has `autoAssign=true` and `defaultAssigneeId`, set `assignedToId` to that user
   - Create TicketActivity record: `{ tenantId, ticketId, actorId, activityType: 'CREATED', metadata: { title, type, priority } }`
   - Return created ticket with includes (assignedTo, requestedBy, queue, category, sla)

   **Export `async function updateTicket(tenantId, ticketId, data, actorId)`**:
   - Fetch existing ticket, compare each changed field
   - For status changes: validate against ALLOWED_TRANSITIONS map, throw `{ statusCode: 400, message: 'Invalid status transition from X to Y' }` if disallowed
   - If status changes to RESOLVED: set `resolvedAt = new Date()`
   - If status changes to CLOSED: set `closedAt = new Date()`
   - If status changes to PENDING: store `slaPausedAt` in customFields JSON (for SLA pause/resume, will be consumed by SLA service in plan 02)
   - If status changes FROM PENDING: calculate pause duration and shift slaBreachAt forward
   - For EACH changed field: create TicketActivity record with `{ tenantId, ticketId, actorId, activityType: 'FIELD_CHANGED', fieldName, oldValue: String(old), newValue: String(new) }`
   - Update ticket with all changed fields
   - Return updated ticket

   **Export `async function addComment(tenantId, ticketId, data, actorId)`**:
   - data includes: `content: string`, `visibility: 'PUBLIC' | 'INTERNAL'`, `timeSpentMinutes?: number`
   - Create TicketComment record
   - Create TicketActivity: `{ activityType: 'COMMENT_ADDED', metadata: { visibility, commentId } }`
   - If this is the first response from a non-requester (agent/admin), set `firstResponseAt` on ticket if null
   - Return created comment

   **Export `async function getTicketList(tenantId, filters)`**:
   - filters: `{ status?, priority?, assignedToId?, categoryId?, queueId?, search?, dateFrom?, dateTo?, page?, pageSize? }`
   - Build Prisma `where` clause: always include `tenantId`
   - For `search`: use `OR: [{ title: { contains: search, mode: 'insensitive' } }, { description: { contains: search, mode: 'insensitive' } }]`
   - Include `assignedTo` (select: id, firstName, lastName, email), `category` (select: id, name), `queue` (select: id, name)
   - Paginate with `skip` and `take` (default pageSize=25, max 100)
   - Also return `total` count for pagination
   - Order by `createdAt desc`

   **Export `async function getTicketDetail(tenantId, ticketId)`**:
   - findUnique with tenantId + id
   - Include: assignedTo, requestedBy, queue, sla, category, comments (include author, ordered by createdAt asc), attachments (include uploadedBy), activities (ordered by createdAt desc, take 50), knowledgeArticles (include knowledgeArticle), cmdbTicketLinks

   **Export `async function assignTicket(tenantId, ticketId, assignedToId, actorId)`**:
   - Update ticket assignedToId, log TicketActivity ASSIGNMENT_CHANGED

   **Export `async function linkKnowledgeArticle(tenantId, ticketId, knowledgeArticleId)`**:
   - Create TicketKnowledgeArticle record (upsert to handle re-link)

   **Export `async function linkCmdbItem(tenantId, ticketId, ciId, linkType = 'AFFECTED')`**:
   - Create CmdbTicketLink record (upsert to handle re-link)

   - Body (JSON): `{ title: string, description?: string, type?: 'INCIDENT'|'SERVICE_REQUEST'|'PROBLEM', priority?: 'LOW'|'MEDIUM'|'HIGH'|'CRITICAL', categoryId?: string, queueId?: string, assignedToId?: string, slaId?: string, tags?: string[] }`
   - Validate with Zod: title minLength 1 maxLength 500, description maxLength 10000
   - Call `createTicket(tenantId, body, userId)`
   - Return 201 with ticket object

   **GET `/api/v1/tickets`** â€” List tickets
   - Query params: status, priority, assignedToId, categoryId, queueId, search, dateFrom, dateTo, page (default 1), pageSize (default 25)
   - Call `getTicketList(tenantId, filters)`
   - Return `{ data: tickets, total, page, pageSize }`

   **GET `/api/v1/tickets/:id`** â€” Get ticket detail
   - Call `getTicketDetail(tenantId, params.id)`
   - Return 404 if not found
   - Return ticket object

   **PATCH `/api/v1/tickets/:id`** â€” Update ticket
   - Body (JSON): any subset of `{ title, description, status, priority, assignedToId, queueId, categoryId, slaId, resolution, tags }`
   - Call `updateTicket(tenantId, params.id, body, userId)`
   - Return updated ticket

   **POST `/api/v1/tickets/:id/comments`** â€” Add comment
   - Body: `{ content: string, visibility?: 'PUBLIC'|'INTERNAL', timeSpentMinutes?: number }`
   - Validate: content minLength 1, timeSpentMinutes positive integer if present
   - If user role is `end_user`, force visibility to `PUBLIC` (end users cannot create INTERNAL comments)
   - Call `addComment(tenantId, params.id, body, userId)`
   - Return 201 with comment

   **POST `/api/v1/tickets/:id/attachments`** â€” Upload attachment
   - Use `request.file()` from @fastify/multipart (scoped registration)
   - Validate: file size <= 25MB, filename present
   - Generate storage key: `${tenantId}/tickets/${ticketId}/${Date.now()}-${filename}`
   - Call `storageService.uploadFile(buffer, key, mimetype)`
   - Create TicketAttachment record: `{ tenantId, ticketId, uploadedById: userId, filename, mimeType, fileSize, storagePath: key }`
   - Create TicketActivity: `{ activityType: 'ATTACHMENT_ADDED', metadata: { filename, fileSize } }`
   - Return 201 with attachment record

   **GET `/api/v1/tickets/:id/attachments/:attachmentId/url`** â€” Get signed download URL
   - Lookup TicketAttachment by id + tenantId
   - Call `storageService.getFileSignedUrl(attachment.storagePath)`
   - Return `{ url }`

   **POST `/api/v1/tickets/:id/assign`** â€” Assign ticket
   - Body: `{ assignedToId: string }`
   - Call `assignTicket(tenantId, params.id, body.assignedToId, userId)`
   - Return updated ticket

   **POST `/api/v1/tickets/:id/link-article`** â€” Link KB article
   - Body: `{ knowledgeArticleId: string }`
